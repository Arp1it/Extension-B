# 🎤🤖 Project: Talking Robot Assistant (Docs for Beginners)

## 🧠 What are we building?

We are making a system where:

* 🎤 You speak into the mic
* 🌐 Your voice goes to a server
* 🤖 Server replies with voice
* 🔊 You hear the reply

👉 Like talking to Alexa / Google Assistant!

---

# 🧩 Full Code Breakdown (Step-by-Step)

---

# 🔌 PART 1: Connecting to the Server

```js
const socket = new WebSocket("ws://127.0.0.1:5000/audio");
```

### 🧠 What is happening?

* We are creating a **WebSocket connection**
* This is like opening a **live phone call** 📞

### 💡 Simple meaning:

* `"ws://..."` → server address
* `127.0.0.1` → your own computer (localhost)
* `5000` → port (like a door number)
* `/audio` → specific route

👉 So we are saying:

> “Hey server, let’s talk in real-time!”

---

# 🧠 PART 2: Memory Variables (Flags)

```js
let speaking = false;
let micStarted = false;
```

### 🗣️ `speaking`

* `false` → assistant is silent
* `true` → assistant is talking

👉 Why needed?
To stop mic while assistant is speaking
(otherwise → infinite echo loop 🔁)

---

### 🎤 `micStarted`

* Prevents mic from starting multiple times

👉 Like:

> “Only one mic allowed 😤”

# 🎛️ PART 7: Audio Processing System

```js
const audioContext = new AudioContext({ sampleRate: 16000 });
```

### 🧠 What is AudioContext?

* A **sound engine** inside browser

### 💡 `16000`:

* How many audio samples per second

👉 Lower = smaller data (good for speech)

---

```js
if (audioContext.state !== "running") {
    await audioContext.resume();
}
```

👉 Fix:

* Some browsers pause audio → we resume it

---

```js
const source = audioContext.createMediaStreamSource(stream);
```

👉 Convert mic stream → usable audio input

---

```js
const processor = audioContext.createScriptProcessor(4096, 1, 1);
```

### 🧠 What is this?

* Breaks audio into chunks

### 💡 Meaning:

* 4096 → size of each chunk
* 1 input → mono audio
* 1 output

👉 Like:

> “Give me small packets of sound again and again”

---

# 🔇 PART 8: Silence Trick (IMPORTANT)

```js
const silence = audioContext.createGain();
silence.gain.value = 0;
```

👉 Create silent audio

---

```js
source.connect(processor);
processor.connect(silence);
silence.connect(audioContext.destination);
```

### 🧠 Flow:

```
Mic → Processor → Silence → Speaker
```

👉 Why?

* Browser requires output connection
* But we don’t want sound to play

👉 So:

> “Process audio but keep it silent”

# 🔥 FINAL FULL FLOW (VERY IMPORTANT)

### Step-by-step:

1. Page opens 🌍
2. WebSocket connects 🔌
3. Mic permission checked 🎤
4. Mic starts recording
5. Audio → sent to server
6. Server processes it
7. Server sends back voice
8. Browser plays it 🔊
9. While playing → mic stops ❌
10. After playing → mic resumes ✅
